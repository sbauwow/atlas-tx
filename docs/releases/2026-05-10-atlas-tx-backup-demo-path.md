# Atlas TX — backup demo path if the live strip test fails

Primary live destination: https://atlastexas.org

Use this if the Android device, strip, camera flow, or sample setup fails in the room.

## Core rule
Do not let the live strip test become the centerpiece of the entire demo.

Atlas TX should still land strongly if the backup path is:
- live site first
- one county / permit / water investigation path second
- MCP + skill third
- app lane shown as a product extension rather than a fragile climax

## Backup demo structure

### 1. Open the live site
Say:
"The product is live at atlastexas.org, and that is the primary experience I want to show first."

### 2. Walk one strong investigation path
Choose one:
- a county map -> county detail path
- a water workflow path
- a permit pressure / filing path

Say:
"Even without the app moment, the core product stands on its own as a county-intelligence evidence system."

### 3. Show the MCP surface
Run one or two commands:
```bash
npm run mcp -- discover_datasets
npm run mcp -- summarize_water_risk_for_county '{"county":"Travis County"}'
```

Say:
"The same evidence stack is available programmatically, so this is not just a visual demo."

### 4. Show the skill file or prepared skill invocation
Point to:
- `skills/atlas-tx/SKILL.md`

Say:
"This is the reusable agent wrapper for the MCP surface."

### 5. Replace the live strip moment with a framed app walkthrough
If the strip test fails, still show:
- Android app home/capture screen
- the fact that it posts to the observation route
- the evidence-separation rule

Say:
"The field-verification lane is part of the roadmap and product surface, but the main story today is that Atlas TX already works as a live web + MCP + skill evidence system."

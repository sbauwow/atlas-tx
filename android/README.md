# Atlas TX Capture (Android)

Companion mobile capture client for the Atlas TX citizen water-strip lane.

Pairs with the existing web flow at `src/app/citizen/` and posts to
`POST /api/citizen/observations` (multipart). All on-device strip colorimetry
is intentionally deferred вЂ” v1 sends a minimal `clientReading` and lets the
server-side QA + vision pass do the reading. Same caveats as the web lane:
**screening only, not regulatory**. See `docs/contracts/community-observation.md`.

The phone never calls an inference API directly and never holds an inference
key. The server route runs the vision pass against
[Featherless](https://featherless.ai) (preferred,
`https://api.featherless.ai/v1`) when `FEATHERLESS_API_KEY` is set, falling
back to OpenAI's `gpt-4o-mini` only when Featherless is unkeyed. Default
Featherless model is `meta-llama/Llama-4-Scout-17B-16E-Instruct`; override
with `FEATHERLESS_MODEL`. The actual model that ran is recorded on each
observation as `llmModel: "featherless:<model>"` (or `"openai:<model>"`).
See `src/lib/observations/vision.ts`.

## Stack

- Kotlin 2.0, Jetpack Compose, Material 3
- minSdk 31 (Android 12), targetSdk 35 (Android 15)
- AGP 8.7, Gradle 8.10
- Camera capture via `MediaStore.ACTION_IMAGE_CAPTURE` (system camera) вЂ” same
  pattern as the web `<input type="file" capture="environment">`.
- DataStore Preferences for anonymous device UUID + base URL
- OkHttp + kotlinx.serialization for the multipart POST

## First build

The wrapper jar is not committed вЂ” generate it once:

```sh
cd android
gradle wrapper --gradle-version 8.10.2
./gradlew assembleDebug
```

Or just open `android/` in Android Studio Hedgehog (or newer); it will
download Gradle and AGP automatically.

The default base URL is `http://10.0.2.2:3000`, which targets the host
machine's `localhost:3000` from the Android emulator. Change it in
**Settings** if you point the app at a deployed Atlas TX instance.

## API contract

Submits multipart/form-data to `POST {baseUrl}/api/citizen/observations`:

| Field | Value |
|---|---|
| `image` | JPEG file (в‰¤ 8 MB) |
| `clientReading` | `{"schemaVersion":1,"chartId":"jed-pool-tools-5way-v1","perAnalyte":[]}` |
| `stripBrand` | from settings, default `JED Pool Tools 5-way` |
| `countySlug` | optional, e.g. `travis-county` |
| `X-Atlas-Capture-Device-Id` | header вЂ” anonymous UUID |

Response shape: `{ observation: {вЂ¦}, deduped?: boolean }`.
